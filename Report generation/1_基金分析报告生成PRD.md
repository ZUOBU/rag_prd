# 基金分析报告生成 PRD（股票型、混合型 - 偏股）（单支）

本功能旨在通过 AI 自动调取数据库中的数据，结合大模型的语义理解能力，为客户（及客户经理）提供一份专业、直观、且具备个性化持仓建议的基金分析报告。

# 一、功能概述

| 名称 | 基金组合助手 - 基金分析报告生成 |
|------|--------------------------------|
| 目标用户 | 零售客户 / 客户经理 / 投顾 |
| 核心能力 | 单基金分析 & 双基金对比分析 |
| 输入 | 基金代码（最多 2 只）、客户 ID（可选） |
| 输出 | 结构化 + 解释性基金分析报告 |
| 核心原则 | 数据驱动 + LLM 总结 + 过程透明 |

# 二、入口

1. 客服 - 服务 tab-深度研究助手 - 基金组合助手 - 基金分析
2. 客服 - 服务 tab-热门服务 - 基金组合助手 - 基金分析
3. 首页 Banner 等

# 三、信息收集

## 1、自然语言触发

用户在客服对话或者基金组合助手对话中命中"特定基金分析（单支）"意图时，路由至基金分析报告生成流程，遵循以下链路：

### **前置校验**

**登录状态校验**。若用户未登录，对话窗口回复如下：

> 您需要先登录才能继续对该基金的分析哟。

并显示登录按钮，用户已输入内容写入 Memory，方便登录后的处理。用户点击后，跳转至登录页面。用户成功登录返回后，流程继续。若用户不登录，则流程中断。

### **槽位提取**

结合改写后的 Query、意图识别结果、长短期记忆 (Memory) 综合提取槽位。槽位如下：

| 字段 | 说明 | 默认值 |
|------|------|--------|
| 基金名称 1 | 必填 | 无 |

## 2、约束性交互

用户通过点击按钮，进入特定基金分析，选择需要分析的基金（最多 2 只）。

- 选择 1 只则进入"特定基金分析（单支）"流程。
- 选择 2 只则进入"特定基金分析（2 只）"流程。

注：用户选择完成后，同样需要**先校验登录状态**。若用户未登录，对话窗口回复如下：

> 已准备开始分析您选择的基金，请先登录哟。

显示登录按钮，用户成功登录返回后，流程继续。

# 四、整体流程设计

## 1、报告生成流程（对用户可见）

| 步骤序号 | 步骤名称 | 展示文案（前端） | 处理内容 |
|----------|----------|------------------|----------|
| 1 | 基金信息校验 | 正在获取基金基础信息 | 校验基金名称，获取代码 |
| 2 | 数据检索 | 正在加载市场与基金数据 | 查询数据库 |
| 3 | 指标计算 | 正在计算收益与风险指标 | Sharpe、回撤等 |
| 4 | 持仓分析 | 正在解析持仓结构 | 行业/股票分布 |
| 5 | 估值分析 | 正在评估基金估值水平 | PE/PB 分位 |
| 6 | 基金经理分析 | 正在分析基金经理能力 | 任期、历史收益 |
| 7 | 智能总结 | 正在生成分析结论 | LLM 生成 |
| 8 | 报告生成 | 正在生成完整报告 | 汇总输出 |

## 2、基金信息校验

若流程由用户自然语言触发，则需要该步骤，使用槽位中的"基金名称"作为关键词在系统"基金名称"中检索，获取基金代码。

匹配成功阈值：文字匹配度 85%，且忽略以下文字的匹配：

```markdown
混合、成长、质量、配置、
```

注：字母不可忽略。若匹配到的结果不唯一，取 top2 反馈给用户选择，反馈话术如下：

```markdown
您需要分析的基金是：
    1、基金名称 1
    2、基金名称 2
```

# 五、单基金分析报告

## 1、简短评价

- **生成方式**：MySQL 查询结果 + LLM 生成
- **输出形式**：150 字以内的概括性评论
- **示例**："您持有的永赢科技智选混合 A(022364) 近期出现小幅回调，但长期收益表现突出，跑赢了同类基金。这只基金属于高风险高收益的科技主题产品，需要结合你的持仓结构综合评估。"
- **结构**：您（已持有 | 已加入自选的 | 关注的）"fund_name"("fund_code")"Continue_brief_evaluation（由 LLM 输出）"。
  - 需根据客户持仓基金、客户自选基金来判断是非为"已持有"、"已加入自选的"（两者均是，只显示"持有的"），如果均不是，统称为"关注的"。

### LLM 输出示例

```json
{
  "Continue_brief_evaluation": "近阶段表现震荡偏强，短期略有回调，但长期收益表现稳健，体现出成长风格基金特征，波动相对较高。",
  "Purchase_Suggestion": "可逢低布局",
  "Suggested_purchase_level": "3",
  "Product_performance": "整体收益表现较好，长期跑赢基准",
  "Industry_analysis": "具备成长空间，短期受市场影响波动大",
  "Holding_relationship": "与现有持仓风格重合，分散效果一般"
}
```

## 2、智能摘要

- **生成方式**：MySQL 查询结果 + LLM 生成
- **输出形式**：信息卡片

### MySQL 查询结果部分

| 字段 | 字段中文名 | 数据来源 |
|------|------------|----------|
| fund_name | 基金名称 | |
| | 客户持有 | 自选状态 | 在【简短评价】中，是否持有/自选已比较得出，如果非"持有"且非"自选"则不显示此字段 |
| fund_code | 基金代码 | |
| fund_type | 基金类型 | |
| fund_risk_level | 基金风险等级 | |

### LLM 输出

| 字段 | 字段中文名 | 说明 |
|------|------------|------|
| **Purchase_Suggestion** | 配置建议摘要 | 解释是否建议配置（**10 个字以内**） |
| **Suggested_purchase_level** | **建议持仓级别** | 用于控制右上角表情的显示。<br>- "4"：很建议增持<br>- "3"：可考虑适当配置<br>- "2"：保持中性<br>- "1"：不建议继续增持 |
| **Product_performance** | 产品表现 | 根据数据作出总结（不展示数据本身）（**18 字以内**） |
| **Industry_analysis** | 行业分析 | 评估该行业长期发展趋势，以及短期波动情况（**18 字以内**） |
| **Holding_relationship** | 与客户持仓关系 | 评估基金与客户持仓的重合度/分散性（**18 字以内**） |

## 3、基金画像（收益 + 风险）

### 3.1、概述

- **生成方式**：LLM 生成，取 prompt 输出中的"Fund_profile"字段
- **Prompt**：3_基金画像 Prompt

#### LLM 输出示例

```json
{
  "Fund_profile": "震荡上行偏成长型",
  "Return_ability_title": "收益表现具备弹性",
  "Return_ability_desc": "整体呈现震荡上行趋势，阶段性回撤后具备一定修复能力，收益弹性较为明显。",
  "Risk_feature_title": "波动水平相对较高",
  "Risk_feature_desc": "净值波动幅度较大，短期存在回撤压力，整体风险特征偏成长型。"
}
```

### 3.2、图表展示

取基金详情页已有的收益走势，单位净值折线图展示。在客户心中形成大概印象，避免连续文本造成报告枯燥。

- 可切换至"单位净值"查看；
- 可切换：近 1 月、近 3 月、近 6 月、近 1 年、成立以来。

### 3.3、收益能力

- **生成方式**：概述文字由 LLM 生成，数据由 MySQL 检索或计算
- **标题**：收益能力："Return_ability_title"（LLM 输出）
- **正文**：截至 "update_time"，该基金（近 1 年 (涨 | 跌) 幅达 | 年化收益率）"return_rate_1y"，超额收益率"Excess_return"(若为跌则无此句)，优于"Better_category_rate"同赛道基金。近 1 年创新高天数"record_days_1y"天，滚动持有"N"天超额胜率"excess_win_rate_N"，"Return_ability_desc(由于 LLM 生成的总结话术)"。

| 字段 | 字段中文名 | 说明 | 是否 LLM 输出 | 展示方式 |
|------|------------|------|---------------|----------|
| Return_ability_title | 收益能力标题 | LLM 输出 | √ | 文字 |
| update_time | 更新时间 | 基金收益数据表中的更新时间 | × | YYYY 年 MM 月 DD 日 |
| return_rate_1y | 近 1 年收益率 | 基金数据库（若为负值，则为跌，取绝对值）<br>若基金成立不满一年，则计算年化收益率（保留正负号） | × | %，保留 2 位小数 |
| Excess_return | 超额收益率 | 实际收益率 (Rp)−基准收益率 (Rb)<br>注意前后时间区间需一致 | × | %，保留 0 位小数 |
| Better_category_rate | 优于同赛道基金 | 同赛道中年（化）收益率小于本基金的支数/赛道基金总支数，弱小于 20% 此句不显示。 | × | %，保留 0 位小数 |
| record_days_1y | 新高天数 | 单位净值高于历史过往净值，则为新高。<br>筛选近 1 年的新高天数总计求和，若为 0 则此句不显示 | × | 整数 |
| excess_win_rate_N | 滚动持有 N 天超额胜率 | 调用 API 查询获得（入参：基金代码、天数）<br>天数默认 180 天，基金成立不满一年时，依次尝试 90 天，30 天，7 天。<br>若以 7 天查询依然无结果，则此句不显示。 | × | %，保留 0 位小数 |
| Return_ability_desc | 收益能力描述 | LLM 输出 | √ | 文本总结 |

### 3.4、风险特征

- **生成方式**：概述文字由 LLM 生成，数据由 MySQL 检索或计算
- **标题**：风险特征："Risk_feature_title"
- **正文**：截至 "deadline_time"，该基金（近 1 年 | 成立以来）最大回撤 "Max_drawdown_1y"，波动率"volatility"，"Risk_feature_desc(由于 LLM 生成的总结话术)"。

| 字段 | 字段中文名 | 说明 | 是否 LLM 输出 | 展示方式 |

# 16. Memory(记忆)体系设计

- **原则**：内外双轨、长短期分离，以保障上下文连贯性、解决合规权限隔离以及提供千人千面的个性化服务。

## 16.1 短期记忆
- **作用**：维护单次会话内多轮对话的连贯性、语境代词消解及Agent操作槽位收集。
- **生命周期**：距上次交互超24小时即清空。
- **字段**：
  - `session_id`: 会话唯一绑定标识。
  - `turn_count`: 当前有效对话交互轮次。
  - `dialogue_history`: 最近5轮的上下文文本流 `[{role: "user/assistant", content: "..."}]`。
  - `active_intent_slots`: 正在执行中且挂起的业务槽位字典（如 `{card_num: null}`）。

## 16.2 长期记忆
- **作用**：存储员工和客户的基础信息，用于支持个性化服务和合规风控。
- **生命周期**：永久保存。静态字段由行内系统定时跑批同步。

### 16.2.1 面向内部员工 (Staff)
- **字段**：
  - `emp_id`: 员工工号。
  - `dept_code`: 隶属业务条线代码。**（可用作不同知识库查询的优先级）。**
  - `job_title`: 职级/岗位（可用作鉴权）。
  - `branch_region`: 归属网点代码（拉取属地特异性版本或政策）。
  - `frequent_kb_areas`: 历史高频查询聚类（用于前端猜你想问等的问题推荐，**待加入**）。

### 16.2.2 面向外部客户 (Customer)
- **字段**：
  - `customer_id`: 客户唯一标识。
  - `age`: 年龄。
  - `gender`: 性别。
  - `residence_city`: 常住地。
  - `cust_tier_level`: 客户等级（N1-S5）。**不同等级客户福利权益不同**
  - `risk_tolerance_level`: 风险承受力等级（R1保守至R5激进）。理财推荐时必要的参考信息。产品风险高于客户风险承受力时，作出必要提醒。
  -  `deposit card`：客户名下储蓄卡卡号，多个时使用逗号隔开。
  - `credit card`：客户名下信用卡卡号，多个时使用逗号隔开。
  - `extracted_preferences` (**依托Vector DB存储**): 大模型从历史会话长文中提取出的非格式化隐式偏好实体。例如：“客户抵触超长期锁仓型理财产品”或“客户偏好投向半导体行业的基金”。（**待加入**）

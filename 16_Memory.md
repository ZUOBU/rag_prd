# 16. Memory(记忆)体系设计

- **原则**：内外双轨、长短期分离，以保障上下文连贯性、解决合规权限隔离以及提供千人千面的个性化服务。

## 16.1 短期记忆
- **作用**：维护单次会话内多轮对话的连贯性、语境代词消解及Agent操作槽位收集。
- **生命周期**：距上次交互超24小时即清空。
- **存储方式**：Redis 缓存
- **字段**：
  - `session_id`: 会话唯一绑定标识。
  - `user_type`: `staff`（内部员工）或 `customer`（外部客户）。权限分流依据。
  - `turn_count`: 当前有效对话交互轮次。
  - `dialogue_history`: 最近5-10轮的上下文文本流 `[{role: "user/assistant", content: "..."}]`。
  - `active_intent_slots`: 正在执行中且挂起的业务槽位字典（如 `{card_num: null}`）。

## 16.2 长期记忆
- **生命周期**：永久保存。静态字段由行内系统定时跑批同步；非通用隐式偏好由大模型定时剥离异步萃取。
- **存储方式**：
  - **静态全量结构化事实现状**：MySQL / PostgreSQL（存储标签、职级权限、资产标尺）。
  - **语义表征动态提取池**：向量数据库 Vector DB（存储从文本流水中异步萃取的非结构化碎片）。

### 16.2.1 面向内部员工 (Staff)
- **字段**：
  - `emp_id`: 员工工号。
  - `dept_code`: 隶属业务条线代码。**（可用作不同知识库查询的优先级）。**
  - `job_title`: 职级/岗位（可用作鉴权）。
  - `branch_region`: 归属网点代码（拉取属地特异性版本或政策）。
  - `frequent_kb_areas`: 历史高频查询聚类（用于前端推荐专栏加载预热，待加入）。

### 16.2.2 面向外部客户 (Customer)
- **核心字段 (综合库同步)**：
  - `customer_id`: 客户唯一标识。
  - `basic_profile`: 静态基础统计信息（包含：`age`年龄、`gender`性别、`residence_city`常住地）。
  - `cust_tier_level`: 客户等级（大众散客、金葵花、私行等）。**（核心作用：强力防幻觉，阻断私行条款外泄给普通长尾客引发系统级客诉风控事故）。**
  - `risk_tolerance_level`: 风险承受力等级（R1保守至R5激进）。生成包含关联高息产品回答时，执行不可退撤的合规兜底。
  - `holdings_tag_flags`: 持仓标记布尔值数组集合（如 `has_mortgage`, `has_credit_card`，不暴露确切资金余额，仅辅导大模型掌握推荐语境）。
  - `extracted_preferences` (依托Vector DB存储): 大模型从历史会话长文中提取降维的非格式化隐式偏好向量实体。例如：“客户抵触超长锁仓型结构产品”或“备注周四不开机拒接电销”。

# 元数据
## Chunk
为每个传入的文档分块(Chunk)打上元数据，以便后续溯源及多维联合检索过滤。
元数据设定如下：
|key|说明| value |
|---|---|---|
|`Doc_ID`|（原文档ID）|系统自动生成、不可重复|
|`Page_Num`|（页码）|通过解析结果提取、提取不到时默认为1|
|`Chunk_ID`|（片段ID）|系统自动生成、不可重复|
|`Block_ID`|（父级ID）|系统自动生成、不可重复、若无则为null|
|`Version`|（版本号）|自动识别文档名称、可修改、若无则为1.0|
|`Release_date`|（发版日期）|自动识别文档名称、可修改、若无则为当前日期|
|`Url`|（原文链接）|自动提取、可修改|
|`Image_url`|（相关图片链接）|自动提取、可修改|
|`Table_url`|（相关表格链接）|自动提取、可修改|
|`function_name`|（关联功能的名称）|自动提取、可修改（关联的功能按钮名）|
|`function_ID`|（关联功能ID）|自动提取、可修改（据此获取跳转链接）|
|`confidentiality`|（保密程度）|自动提取、可修改（公开、行内、机密）|

## QA库
针对QA库，元数据设定如下：

### Question
|key|说明| value |
|---|---|---|
|`Question_ID`|（问题ID）|系统自动生成，不可重复|
|`Doc_ID`|（原文档ID）|读取文档ID|
|`Version`|（版本号）|自动识别文档名称、可修改|
|`Release_date`|（发版日期）|自动识别文档名称、可修改|

### 相识问
|key|说明| value |
|---|---|---|
|`Question_ID`|（问题ID）|需要与标准问ID保持一致|

### Answer
|key|说明| value |
|---|---|---|
|`Question_ID`|（问题ID）|读取关联问题ID|
|`Doc_ID`|（原文档ID）|读取文档ID|
|`Chunk_ID`|（片段ID）|系统自动生成|
|`Version`|（版本号）|自动识别文档名称、可修改、若无则为1.0|
|`Release_date`|（发版日期）|自动识别文档名称、可修改、若无则为当前日期|
|`Url`|（原文链接）|自动提取、可修改|
|`Image_url`|（相关图片链接）|自动提取、可修改|
|`Table_url`|（相关表格链接）|自动提取、可修改|
|`function_name`|（关联功能的名称）|自动提取、可修改（关联的功能按钮名）|
|`function_ID`|（关联功能ID）|自动提取、可修改（据此获取跳转链接）|
|`confidentiality`|（保密程度）|自动提取、可修改（公开、行内、机密）|

# 18. 工具清单 (Function Call)

本清单用于枚举客服助手已规划的各项可调用工具（Function Call）。

系统前端（多渠道）以及后续的 MCP（Model Context Protocol）统一挂载层接口，可基于此清单定义实现结构化的方法调用。

> **前端展示策略说明**：
> 针对工具执行结果，我们将设定两种前端接管机制：
> 1. **由大模型再生成**：对于简单的状态操作（如成功/失败/查数字），可将出参信息传递给大模型，由 LLM 生成自然语言回复。
> 2. **前端卡片显示**：对于列表、账单明细、排队号单等复杂多维信息，大模型仅做一句口语引导，具体的出参数据由前端解析 JSON 原生渲染为**富文本交互卡片**。

## 1. 短信服务状态查询 (`query_sms_service_status`)
**功能描述**：查询指定银行帐号或信用卡的动账短信提醒服务状态。
**调用入参**：
| 字段名 (Field) | 字段含义 | 数据类型 | 必填 | 默认值 | 描述说明 |
|---|---|---|---|---|---|
| `account_number` | 账号/卡号 | varchar(30) | 是 | 无 | 需查询短信服务的储蓄卡或信用卡号 |

如果读取到多张`account_number`，需要用户二次确认。
- 例如：请问需要查询哪张卡的短信服务状态？
    - 1.储蓄卡 6222***********0123
    - 2.信用卡 6222***********8901
    - 以上都要

**接口出参**：
| 字段名 (Field) | 字段含义 | 数据类型 | 描述说明 |
|---|---|---|---|
| `account_number` | 账号/卡号 | varchar(30) | 需查询短信服务的储蓄卡或信用卡号 |
| `sms_service_status` | 短信服务状态 | varchar(20) | 枚举：`ENABLE`(开启) / `DISABLE`(关闭) |
| `phone_number` | 接收手机号 | varchar(20) | 接收短信的手机号 |

**后续展示策略**：
1. 仅查询单卡：**固定话术模板**。例如：“您尾号为1234的储蓄卡已开通动账短信服务，接收手机号为138****5678”。
2. 查询多卡：**前端卡片显示**。渲染一张“短信服务状态查询”卡片，包含卡号、短信服务状态、接收手机号3个字段。

## 2. 短信服务开关 (`sms_service_switch`)
**功能描述**：开启或关闭指定银行帐号或信用卡的动账短信提醒服务。
**调用入参**：
| 字段名 (Field) | 字段含义 | 数据类型 | 必填 | 默认值 | 描述说明 |
|---|---|---|---|---|---|
| `account_number` | 账号/卡号 | varchar(30) | 是 | 无 | 需开通/关闭短信服务的储蓄卡或信用卡号 |
| `action` | 操作类型 | varchar(10) | 是 | 无 | 枚举：`enable`(开启) / `disable`(关闭) |
| `phone_number`| 接收手机号 | varchar(20) | 否 | 用户预留手机 | 接收短信的手机号，若为空则读取核心绑定的原预留号码 |
| `verify_code` | 短信验证码 | varchar(10) | 是 | 无 | 通过接口发送至新手机号的数字验证码 |

如果读取到多张`account_number`，需要用户二次确认（单选）。
- 例如：请问需要查询哪张卡的短信服务状态？
    - 1.储蓄卡 6222***********0123
    - 2.信用卡 6222***********8901

**接口出参**：
| 字段名 (Field) | 字段含义 | 数据类型 | 描述说明 |
|---|---|---|---|
| `account_number` | 账号/卡号 | varchar(30) | 需开通/关闭短信服务的储蓄卡或信用卡号 |
| `operation_result` | 操作结果 | varchar(20) | 枚举：`success`(成功) / `failed`(失败) |
| `sms_service_status` | 短信服务状态 | varchar(20) | 枚举：`enable`(开启) / `disable`(关闭) |
| `phone_number` | 接收手机号 | varchar(20) | 接收短信的手机号 |

**后续展示策略**：**固定话术模板**。
- 如果 `operation_result` 为 `success`，例如：“您尾号1234的储蓄卡/信用卡已成功开通/关闭动账短信，接收手机号为138****5678”。
- 如果 `operation_result` 为 `failed`，例如：“抱歉，变更操作失败了呢，要不稍后试试？”。

## 4. 转账限额申请 (`apply_transfer_limit`)
**功能描述**：向风控中心申请临时或永久调整手机银行/网银的非柜面转账交易限额。
**调用入参**：
| 字段名 (Field) | 字段含义 | 数据类型 | 必填 | 默认值 | 描述说明 |
|---|---|---|---|---|---|
| `account_number` | 账户/卡号 | varchar(30) | 是 | 无 | 需要申请调整限额的银行账号 |
| `limit_type` | 限额类型 | varchar(20) | 是 | `daily` | 枚举：`daily`(单日限额) / `single`(单笔限额) / `yearly`(年度限额) |
| `new_limit_amount`| 申请目标额度 | decimal(15,2) | 是 | 无 | 客户期望上调或下调到的具体人民币金额 |
| `face_verify_result` | 人脸核验结果 | varchar(20) | 是 | 无 | 传入人脸核验结果进行核对 |
未获取到`face_verify_result` ，需要引导用户进行人脸核验。


**接口出参**：
| 字段名 (Field) | 字段含义 | 数据类型 | 描述说明 |
|---|---|---|---|
| `ticket_id` | 申请工单号 | varchar(64) | 生成的风控人工复核流水号 |
| `audit_status` | 审核状态 | varchar(20) | 枚举：`PENDING`(审核中) / `APPROVED`(秒批通过) / `REJECTED` |
| `current_limit`| 当前额度 | decimal(15,2) | 调整后的确认额度 |


**后续展示策略**：🟦 **前端卡片显示**。渲染一张“业务受理回执”卡片，包含工单号和进度条，大模型不需转述冗长的工单号。

## 4. 账户余额查询 (`query_account_balance`)
**功能描述**：查询指定借记卡活期、定期或者信用卡可用额度的分布状态。
**调用入参**：
| 字段名 (Field) | 字段含义 | 数据类型 | 必填 | 默认值 | 描述说明 |
|---|---|---|---|---|---|
| `account_number`| 账号/卡号 | varchar(30) | 否 | 默认主卡 | 需要查询的银行账号；空值兜底查默认一类卡 |
| `currency` | 币种 | varchar(10) | 否 | `CNY` | 枚举：`CNY`(人民币) / `USD`(美元) / `EUR`(欧元) |

**接口出参**：
| 字段名 (Field) | 字段含义 | 数据类型 | 描述说明 |
|---|---|---|---|
| `balance_total` | 总余额 | decimal(15,2) | 存款总计或卡片总额度 |
| `usable_balance`| 可用余额 | decimal(15,2) | 扣除冻结后的真实可用金额 |
| `currency` | 结算币种 | varchar(10) | 本位币标识 |

**后续展示策略**：🎯 **由大模型再生成**。将数字带入，以类似“您的借记卡中人民币可用余额为 xx.xx 元”的亲切形式作答。

## 5. 交易明细查询 (`query_transaction_history`)
**功能描述**：查询历史动账明细、工资流水、转账或跨行消费记录。
**调用入参**：
| 字段名 (Field) | 字段含义 | 数据类型 | 必填 | 默认值 | 描述说明 |
|---|---|---|---|---|---|
| `account_number`| 账号 | varchar(30) | 否 | 默认主卡 | 需查询的流水资金账号 |
| `start_date` | 开始日期 | date | 是 | 前置7天 | 搜索跨度的起始点格式为 `YYYY-MM-DD` |
| `end_date` | 结束日期 | date | 是 | 今日当天 | 搜索跨度的截至点格式为 `YYYY-MM-DD` |
| `trans_type` | 交易方向 | varchar(20) | 否 | `all` | 枚举：`all`(全部) / `income`(收入) / `expense`(支出) |
| `keyword` | 对手关键词 | varchar(100)| 否 | `null` | 查询附言或对手方户名的筛选词（如“工资”） |

**接口出参**：
| 字段名 (Field) | 字段含义 | 数据类型 | 描述说明 |
|---|---|---|---|
| `total_count` | 命中笔数 | int | 符合条件的记录总行数 |
| `records` | 明细报文 | json / longtext | 嵌套列表，包含每笔的流水号、时间、对手方、变动金额等 |

**后续展示策略**：🟦 **前端卡片显示**。由前端解析返回的 json 数组，直接呈现在对话气泡内的“下拉账单流卡片”。模型仅用一句话总结：“已为您找到近7天的3笔交易”。

## 6. 信用卡账单与还款 (`credit_card_repayment`)
**功能描述**：拉取客户信用卡本期账单，并执行一键全额、最低还款或自定义扣款等指令。
**调用入参**：
| 字段名 (Field) | 字段含义 | 数据类型 | 必填 | 默认值 | 描述说明 |
|---|---|---|---|---|---|
| `credit_card_no`| 信用卡卡号 | varchar(30) | 是 | 无 | 需要执行还款清欠的信用卡号 |
| `debit_card_no` | 扣款储蓄卡号 | varchar(30) | 否 | 行内存款第一主卡 | 绑定用于支付还款金的自家借记卡号 |
| `repayment_type`| 还款方式 | varchar(20) | 是 | `full` | 枚举：`full`(全额还款) / `minimum`(最低还款) / `custom`(自定义) |
| `amount` | 自定义金额 | decimal(15,2) | 否 | `null` | 选择了 `custom` 时要求传入的还款数字 |

**接口出参**：
| 字段名 (Field) | 字段含义 | 数据类型 | 描述说明 |
|---|---|---|---|
| `repay_status` | 扣款状态 | varchar(20) | 枚举：`SUCCESS` / `INSUFFICIENT_FUNDS`(余额不足等) |
| `actual_repay` | 实扣金额 | decimal(15,2) | 核心账务处理结果金额 |
| `receipt_no` | 还款回单号 | varchar(64) | 凭证单号 |

**后续展示策略**：🟦 **前端卡片显示**。渲染带有电子印章或回执标识的缴款成功卡片，包含明细流水。

## 7. 银行卡挂失冻结 (`report_loss_card`)
**功能描述**：当客户丢卡或遭遇伪冒盗刷时，系统自动挂载该接口执行临时口头挂失或冻结支付动作。
**调用入参**：
| 字段名 (Field) | 字段含义 | 数据类型 | 必填 | 默认值 | 描述说明 |
|---|---|---|---|---|---|
| `card_number` | 尾号/卡号 | varchar(30) | 是 | 无 | 需止付的受损卡片，最少需精确提供后四位 |
| `loss_level` | 止付级别 | varchar(20) | 否 | `temp_lock` | 枚举：`temp_lock`(临时口头) / `permanent_loss`(彻底挂失) |

**接口出参**：
| 字段名 (Field) | 字段含义 | 数据类型 | 描述说明 |
|---|---|---|---|
| `freeze_status` | 冻结状态 | varchar(20) | `SUCCESS` / `FAILED` |
| `expire_time` | 自动解卡时间| datetime | 如果是临时挂失（如预定冻结5天后解除），下发解除时间 |

**后续展示策略**：🎯 **由大模型再生成**。须安抚客户情绪并进行解压提示：“您的储蓄卡已成功口头挂失，资金已冻结。挂失将于 xx 分自动解除，请尽快携证件前往网点补办实体卡。”

## 8. 理财产品购买 (`purchase_wealth_product`)
**功能描述**：发起特定理财产品或代销基金的申购交易动作（调用前必须由 MCP 网关通过了风控前置校验与适当性测评）。
**调用入参**：
| 字段名 (Field) | 字段含义 | 数据类型 | 必填 | 默认值 | 描述说明 |
|---|---|---|---|---|---|
| `product_code` | 产品代码 | varchar(30) | 是 | 无 | 指定基金或理财产品的全行统一代码（如 `008123`） |
| `amount` | 申购金额 | decimal(15,2) | 是 | 无 | 用户期望申购的总金额，校验格式 |
| `payment_account`| 付款卡号 | varchar(30) | 否 | 默认活期投资主账户 | 扣款账户 |

| 字段名 (Field) | 字段含义 | 数据类型 | 描述说明 |

| `status` | 受理状态 | varchar(20) | 枚举：`ACCEPTED`(受理成功待确认) |

## 9. 预约网点排号办理 (`book_branch_appointment`)
**功能描述**：因面签、开立大额存单等必须线下代办的业务，为客户在网点柜台自动取号并预约时段。
**调用入参**：
| 字段名 (Field) | 字段含义 | 数据类型 | 必填 | 默认值 | 描述说明 |
|---|---|---|---|---|---|
| `branch_name` | 目标网点名 | varchar(100) | 是 | 距离最近支行 | 客户意向去的网点行名模糊文本 |
| `biz_type` | 业务大类 | varchar(30) | 是 | `general` | 枚举：`cash`(大额) / `corporate`(对公) / `general`(综合) |
| `book_date` | 预约面签日期| date | 是 | 明日 | 转化后的表单日期字符串 `YYYY-MM-DD` |
| `book_time_slot`| 预约时间段 | varchar(20) | 否 | `morning` | 枚举：`morning` / `afternoon` |

**接口出参**：
| 字段名 (Field) | 字段含义 | 数据类型 | 描述说明 |
|---|---|---|---|
| `appointment_id`| 预约唯一号 | varchar(64) | 预约编号或现场取号的二维码/数字号 |
| `queue_number` | 柜台排队号 | varchar(20) | 例如 `A014` |

**后续展示策略**：🟦 **前端卡片显示**。生成类似登机牌一样的“预约取号票根”，包含排队号码以及可以直接一键拉起地图 App 的地址模块。

## 10. 风险承受能力评估更新 (`update_risk_assessment`)
